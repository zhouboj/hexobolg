Mysql数据库的基本操作应用

发表于 2016-07-19   |   分类于 术业专攻   |     |   阅读次数

Mysql是最流行的关系型数据库管理系统,在WEB应用方面MySQL是最好的RDBMS(Relational Database Management System:关系数据库管理系统)应用软件之一。

Mysql数据库的基本操作

1、显示数据库

  • 语法格式：SHOW DATABASES;
  • 说明：查看你的Mysql服务器上多少个数据库

2、创建数据库存

  • 语法：CREATE DATABASE db_name [IF NOT EXITSTS][CHARSET utf8]
  • 参数：
    • db_name:自己定义的数据库名称。命名规则：字母、数字、下划线.不能是纯数字命名。
    • 【IF NOT EXITSTS】：如果创建的数据库已经存在，则不会报错。
    • 【CHARSET utf8】：指定当前数据库的字符集，默认为latin1。这里的”utf8”不加横线

SHOW 命令的使用
SHOW CREATE DATABASE db_name; //查看创建数据库的语句过程

3、删除数据库

  • 语法格式：DROP DATABASE db_name[IF EXISTS];
  • 参数说明：
    • 【IF EXISTS】条件判断，可选项，如果数据库存在，再进行删除，不会报错。

Mysql 数据库字符集

  • 描述：通过MYSQL命令来修改数据库的字符集
  • 语法: ALTER DATABASE db_name DEFAULT CHARACTER SET utf8;

MySQL 数据表操作

1、选择当前操作的数据库

  • 语法：USE db_name;
  • 说明：选中数据库

2、显示数据表

  • 语法：SHOW TABLES;

3、创建数据表

  • 语法：CREATE TABLE table_name(列1 类型 列的属性，列2 类型 列的属性，……);
  • 说明：多个字段中间用英文下的逗号（,）隔开。
  • 参数说明：
    • 列的名称：命名规则与数据库的规则一样。
    • 列的类型：整形、浮点型、字符型、文本型、日期型
    • 列的属性
      • 列是否为空：NOT NULL | NULL
      • 自动增长：AUTO_INCREMENT
        • 注意：自动增长属性，只能用于id字段。一个表只能有一个自动增长属性。
      • 默认值：DEFAULE value .value的值可以是整形或字符型。
      • 主键（主索引）:PRIMARY KEY.
        • 注意：主索引一般指定是id字段。一个表只能有一个主索引。
      • 结论：id字段的属性，必须有三个：not null、auto increment、primary kdy.

实例：创建一个news的数据表

4、显示数据表

  • 语法：SHOW CREATE TABLE table_name;

5、describe显示表的结构

  • 语法：DESCRIBE table_name;

6、删除数据表

  • 语法：DROP TABLE table_name;

MySQL数据类型

1、整形

类形
tinyint 0-255 1个字节长
smallint 0-65535 2个字节长
int 0-21亿 4个字节长
bigint 0-42亿 8个字节长

2、浮点型

float(M,D) 单精度浮点数。可以精确到小数点后7位。M代表总宽度，D代表小数位。

float(8,2)含义：宽度为8，小数为2，整数为5位。

double(M,D) 双精度浮点数。可以精确到小数点后15位。M代表总宽度，D代表小数位。

3、字符型和文本型

char(M) 固定长度字符，M 代表长度。
varchar(M) 可变长度字符，M 代表长度

char(6) 存储『邮编』。只能是6个
varchar(6) 最少1个，最多6个。

简单分析：char()存取数据的速度，要比varchar()快一点

类型
tinytext 0-255 最小文本型
text 0-1677万 中型文本
longtext 0-42亿 大型文本
